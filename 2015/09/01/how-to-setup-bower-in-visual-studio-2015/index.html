How to setup Bower in Visual Studio 2015

01 set 2015

  • Visual Studio
  • Web

In my previous article I've compared NuGet and Bower, explaining where we can use each one in order to make use of their advantages.

Prior notice

Visual Studio 2015 combined with ASP.NET 5 templates already have Bower integrated by default, but in this example I'm assuming we are developing an ASP.NET 4.6 + MVC 5 web application.

Bower depends on npm. What's that? Well, that's another package manager, but let's just say that it will help us install Bower and leave it there. To do that, we'll need to open a Command Prompt on the same folder as our project. If you don't want to leave Visual Studio, you can make use of the Package Manager Console instead. If you didn't already installed Git for Windows, you'll have to, because Bower let's you install packages directly from a Git endpoint. Go to Git for Windows and install.

Hands-on

Start by installing Bower, typing:

npm install -g bower

That will install Bower on a global location so that it will become available at the command line. It still didn't added anything to the project. Before adding packages, let's create a bower.json file where we'll save all the packages we install in our project (it works just like the packages.config for NuGet). If you're using the Package Manager Console, add a new item to the project and select Bower Configuration File (inside Web > General).

If you're using the Command Prompt, you can add the same file by typing (the command won't work on the Package Manager Console):

bower init

After typing the command above, Bower will ask for information so it can create a bower.json file accordingly to the needs of your project. If you don't know what to choose in any of the questions, press Enter and let Bower fill in the default values, which should be fine.

Adding packages

Now we're ready to add packages to our project. Bower let's you search for packages in the command line, but I rather use Bower's website: http://bower.io/search/

Once you know what to install, you can type:

bower install jquery --save

The --save argument will add the package to the bower.json, so later we can update the package or reinstall the missing packages, just like NuGet does.

The power of Visual Studio shines if you want to avoid the Command Prompt anytime you need to add one more package. Instead, you can install packages by opening the bower.json file and adding the package to the dependencies node.

As soon as you start typing the name of the package, Visual Studio will search it for you and present the available options intellisense style. You'll get the same behavior for the version.
